Read-only audit of the whole sk* ecosystem with the SKSecurity honest-claims
gate (`sksecurity claims scan`). Goal: find **real** security overclaims —
"quantum-proof", "quantum-safe", "unbreakable", "uncrackable", "100% secure",
"military-grade" (as a security claim) — and separate them from honest
negations, quoted/meta references, proper nouns, build artifacts, and test
fixtures.

- **Tool:** `sksecurity.cli claims scan <path> --format json`

- **Crypto honesty standard:** never "quantum-proof"; a classical KEM stays
  harvest-now-decrypt-later (HNDL) exposed forever; a hybrid is only as strong
  as its *strongest surviving leg*; cite FIPS 203 (ML-KEM) / 204 (ML-DSA) /
  205 (SLH-DSA), not the adjective.
- **Scope of edits:** fixes/allowlist **only** inside `sksecurity`. Findings in
  other repos are **flagged here, not edited.**

## Scanner improvements made (in sksecurity, this audit)

Two additive, TDD-backed changes to `honest_claims.py` removed 109 false
positives that were drowning the signal (96→3 in sk-pqc-rs, 8→0 in
sk-pqc-dart, 6→0 in sk-pqc-py, 2→0 in the skpqc site repo):

1. **`target/` added to `SKIP_DIRS`** — the Rust/cargo build directory (peer of
   the already-skipped `dist`/`build`). Compiled `.rlib`/dep blobs were being
   string-scanned and matching on embedded literals. `target/` is gitignored in
   every Rust repo here, so this only ever skips untracked build output.
2. **"Open Quantum Safe" proper-noun exception** — a `quantum-safe` hit that is
   the tail of the project name "Open Quantum Safe" / "open-quantum-safe" (the
   OQS project / liboqs, i.e. immediately preceded by `open` + a separator) is a
   reference, not a claim. Genuine `quantum-safe` claims still fire.

Tests added in `tests/test_honest_claims.py`
(`TestOpenQuantumSafeProperNoun`, `TestCargoTargetSkipped`); full suite **35
passed**. `sksecurity` itself scans **clean (0 findings)**.

### capauth — `docs/CRYPTO_SPEC.md:502` · `quantum-safe` · CRITICAL
```

```
**Why it's an overclaim:** "quantum-safe" asserts a proven guarantee no scheme
gives. The root being *built from* SLH-DSA (FIPS 205) / a hybrid is
**quantum-resistant**, not "safe": a hybrid degrades to whichever leg survives.
**Honest rewrite:**
> Phase 2 means generating a **quantum-resistant** root (SLH-DSA per FIPS 205,
> or a hybrid that holds as long as one leg survives) and a …
